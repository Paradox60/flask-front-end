Работа с admin.py в Django

Полное руководство по настройке и кастомизации административной панели Django

Что такое admin.py?

admin.py - это файл в приложении Django, который отвечает за настройку административной панели. Здесь регистрируются модели, настраивается внешний вид и поведение интерфейса администрирования.

Админка Django - это мощный встроенный инструмент для управления содержимым сайта без необходимости писать собственные интерфейсы.

Базовая настройка административной панели

1

Создание суперпользователя

Перед использованием админки необходимо создать учетную запись суперпользователя:

# В командной строке
python manage.py createsuperuser
2

Регистрация моделей в admin.py

Чтобы модель отображалась в админке, ее нужно зарегистрировать:

# myapp/admin.py
from django.contrib import admin
from .models import MyModel

# Базовая регистрация
admin.site.register(MyModel)
3

Доступ к админке

После регистрации моделей можно зайти в админку по адресу:

http://localhost:8000/admin/

Кастомизация отображения моделей

Для тонкой настройки отображения модели в админке используется класс ModelAdmin.

# myapp/admin.py
from django.contrib import admin
from .models import Article, Category, Author

class ArticleAdmin(admin.ModelAdmin):
    list_display = ['title', 'author', 'category', 'published_date', 'is_published']
    list_filter = ['category', 'published_date', 'is_published']
    search_fields = ['title', 'content']
    list_editable = ['is_published']
    prepopulated_fields = {"slug": ("title",)}
    date_hierarchy = 'published_date'

class CategoryAdmin(admin.ModelAdmin):
    prepopulated_fields = {"slug": ("name",)}

# Регистрация моделей с кастомизацией
admin.site.register(Article, ArticleAdmin)
admin.site.register(Category, CategoryAdmin)
admin.site.register(Author)

list_display

Определяет, какие поля отображать в списке объектов

list_filter

Добавляет фильтры в правую панель для указанных полей

search_fields

Включает поиск по указанным полям

list_editable

Позволяет редактировать поля прямо из списка

Расширенные возможности

1

Inline-модели

Позволяют редактировать связанные модели на той же странице:

class CommentInline(admin.TabularInline):
    model = Comment
    extra = 1 # Количество пустых форм

class ArticleAdmin(admin.ModelAdmin):
    inlines = [CommentInline]
2

Кастомные шаблоны

Можно переопределять шаблоны админки для полной кастомизации:

class ArticleAdmin(admin.ModelAdmin):
    # Использование кастомного шаблона изменения
    change_form_template = 'admin/myapp/article/change_form.html'
3

Переопределение методов

Можно изменять логику сохранения, отображения и других операций:

class ArticleAdmin(admin.ModelAdmin):
    def save_model(self, request, obj, form, change):
        # Автоматическое установление автора
        if not change: # Если объект создается
            obj.author = request.user
        super().save_model(request, obj, form, change)

Полезные советы

Совет 1: Всегда используйте verbose_name и verbose_name_plural в моделях для красивого отображения в админке.

Совет 2: Для полей с большим количеством текста используйте TextareaWidget.

Совет 3: Регулярно создавайте резервные копии базы данных перед массовыми операциями.

Совет 4: Используйте list_per_page для управления количеством объектов на странице.
