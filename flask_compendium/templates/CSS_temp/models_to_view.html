Основной принцип работы

В Django View (представление) - это функция или класс, который обрабатывает запрос и возвращает ответ. Для работы с данными из моделей используются методы QuerySet API.

Модель для примеров

Рассмотрим работу с данными на примере модели BlogPost:

# models.py
from django.db import models
from django.contrib.auth.models import User

class BlogPost(models.Model):
    title = models.CharField("Заголовок", max_length=200)
    content = models.TextField("Содержание")
    author = models.ForeignKey(User, on_delete=models.CASCADE)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    is_published = models.BooleanField(default=False)
    category = models.CharField(max_length=100, default="general")

    def __str__(self):
        return self.title

Function-Based Views (FBV)

Классический подход с использованием функций:

Все записи
Одна запись
Фильтрация
# views.py
from django.shortcuts import render
from .models import BlogPost

def post_list(request):
    # Получаем все опубликованные посты
    posts = BlogPost.objects.filter(is_published=True)
    return render(request, 'blog/post_list.html', {'posts': posts})
# urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('posts/', views.post_list, name='post_list'),
]
def post_detail(request, post_id):
    try:
        # Получаем один пост по ID
        post = BlogPost.objects.get(id=post_id, is_published=True)
    except BlogPost.DoesNotExist:
        # Если пост не найден - 404 ошибка
        from django.http import Http404
        raise Http404("Пост не найден")
    
    return render(request, 'blog/post_detail.html', {'post': post})
# Более простой способ с get_object_or_404
from django.shortcuts import get_object_or_404

def post_detail(request, post_id):
    post = get_object_or_404(BlogPost, id=post_id, is_published=True)
    return render(request, 'blog/post_detail.html', {'post': post})
def posts_by_category(request, category_name):
    # Фильтрация по категории
    posts = BlogPost.objects.filter(
        category=category_name,
        is_published=True
    ).order_by('-created_at') # Сортировка по дате
    
    return render(request, 'blog/category.html', {
        'posts': posts,
        'category': category_name
    })

Class-Based Views (CBV)

Более современный подход с использованием классов:

📋ListView

Для отображения списка объектов

from django.views.generic import ListView
from .models import BlogPost

class PostListView(ListView):
    model = BlogPost
    template_name = 'blog/post_list.html'
    context_object_name = 'posts'
    queryset = BlogPost.objects.filter(is_published=True)
🔍DetailView

Для отображения одного объекта

from django.views.generic import DetailView

class PostDetailView(DetailView):
    model = BlogPost
    template_name = 'blog/post_detail.html'
    context_object_name = 'post'
    
    def get_queryset(self):
        return BlogPost.objects.filter(is_published=True)

Основные методы QuerySet

Методы для работы с данными в Django ORM:

✅all()

Получить все объекты модели

posts = BlogPost.objects.all()
🔍filter()

Фильтрация по условиям

published_posts = BlogPost.objects.filter(
    is_published=True,
    category="django"
)
🚫exclude()

Исключение по условиям

posts = BlogPost.objects.exclude(
    category="archive"
)
📊get()

Получить один объект (вызывает исключение если не найден или найдено несколько)

post = BlogPost.objects.get(id=1)
📈order_by()

Сортировка результатов

posts = BlogPost.objects.order_by(
    '-created_at', # по убыванию
    'title' # по возрастанию
)
🔗select_related()

Оптимизация запросов для ForeignKey

posts = BlogPost.objects.select_related(
    'author' # Загружаем данные автора одним запросом
)

Работа с формами и данными

Пример обработки POST-запросов и создания новых объектов:

from django.shortcuts import render, redirect
from .forms import BlogPostForm
from .models import BlogPost

def create_post(request):
    if request.method == 'POST':
        form = BlogPostForm(request.POST)
        if form.is_valid():
            # Сохраняем данные в базу
            post = form.save(commit=False)
            post.author = request.user
            post.save()
            return redirect('post_detail', post_id=post.id)
    else:
        form = BlogPostForm()
    
    return render(request, 'blog/create_post.html', {'form': form})

Полезные советы и лучшие практики

Оптимизация запросов: Используйте select_related() и prefetch_related() для уменьшения количества SQL-запросов.

Безопасность: Всегда проверяйте права доступа при работе с пользовательскими данными.

Обработка ошибок: Используйте get_object_or_404() вместо try/except для упрощения кода.

Пагинация: Для больших списков используйте Django's Paginator или ListView.paginate_by.
