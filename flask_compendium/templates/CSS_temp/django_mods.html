Django Models

Основы работы с моделями в Django ORM

Что такое Django Models?

Модели в Django - это Python-классы, которые определяют структуру данных вашего приложения. Каждая модель соответствует таблице в базе данных, а атрибуты модели соответствуют полям таблицы.

Примечание: Django ORM (Object-Relational Mapping) автоматически преобразует операции с Python-объектами в SQL-запросы к базе данных.

Создание базовой модели

Модели определяются в файле models.py вашего приложения. Вот пример простой модели:

from django.db import models
from django.urls import reverse

class BlogPost(models.Model):
    title = models.CharField("Заголовок", max_length=200)
    content = models.TextField("Содержание")
    is_published = models.BooleanField("Опубликовано", default=False)
    created_at = models.DateTimeField("Дата создания", auto_now_add=True)
    updated_at = models.DateTimeField("Дата обновления", auto_now=True)

    class Meta:
        verbose_name = "Пост блога"
        verbose_name_plural = "Посты блога"
        ordering = ['-created_at']

    def __str__(self):
        return self.title

    def get_absolute_url(self):
        return reverse('post_detail', kwargs={'pk': self.pk})

Основные типы полей

Текстовые поля

  • CharField - для короткого текста (требует max_length)
  • TextField - для длинного текста
  • EmailField - для email-адресов

Числовые поля

  • IntegerField - целые числа
  • DecimalField - числа с фиксированной точностью
  • FloatField - числа с плавающей точкой

Дата и время

  • DateTimeField - дата и время
  • DateField - только дата
  • TimeField - только время

Другие полезные поля

  • BooleanField - true/false
  • ForeignKey - связь "многие к одному"
  • ManyToManyField - связь "многие ко многим"
  • FileField - для загрузки файлов
  • ImageField - для загрузки изображений

Миграции

После создания или изменения моделей необходимо создать и применить миграции:

# Создание миграций
python manage.py makemigrations

# Применение миграций
python manage.py migrate

# Просмотр SQL-кода миграции
python manage.py sqlmigrate app_name migration_number

Работа с данными (CRUD)

Create - Создание

# Способ 1
post = BlogPost(title="Мой первый пост", content="Содержание...")
post.save()

# Способ 2
post = BlogPost.objects.create(
    title="Мой первый пост",
    content="Содержание..."
)

Read - Чтение

# Получить все объекты
all_posts = BlogPost.objects.all()

# Получить один объект по ID
post = BlogPost.objects.get(id=1)

# Фильтрация
published_posts = BlogPost.objects.filter(is_published=True)

# Сортировка
recent_posts = BlogPost.objects.all().order_by('-created_at')

Update - Обновление

post = BlogPost.objects.get(id=1)
post.title = "Обновленный заголовок"
post.save()

Delete - Удаление

post = BlogPost.objects.get(id=1)
post.delete()

Лучшие практики

  • Всегда добавляйте __str__ метод для читаемого отображения объектов
  • Используйте verbose_name для полей и моделей для админ-панели
  • Определяйте ordering в классе Meta для порядка по умолчанию
  • Используйте related_name для обратных связей ForeignKey
  • Добавляйте индексы для часто используемых полей фильтрации
  • Регулярно делайте миграции при изменении моделей
